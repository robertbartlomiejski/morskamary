# Security Policy

## Supported Versions

Security updates are provided for the current codebase on the default branch
and for the latest published release. Older versions are not actively
maintained for security fixes.

| Version | Supported |
| ------- | --------- |
| latest (`main`) | :white_check_mark: |
| latest release tag | :white_check_mark: |
| older releases | :x: |

## Reporting a Vulnerability

Please report vulnerabilities privately using GitHub's security advisory
workflow for this repository (Security tab → Report a vulnerability).

When submitting a report, include:
- affected file(s) and function(s),
- a clear reproduction path or proof-of-concept,
- potential impact and any suggested remediation.

Expected process:
- initial triage confirmation within 7 days,
- coordinated remediation and disclosure once a fix is ready,
- credit in release notes when desired by the reporter.
